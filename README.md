СИСТЕМА РОЗНИЧНОГО ТОРГОВОГО КОНТРОЛЯ
Версия №-0.0.1
### 1️⃣ Acceptance Service

ОТВЕТСТВЕННОСТЬ:
	1. Прием и обработка сигнала с сканера;
	2. Прием данных от пользователя - название товара/цена закупки/цена реализации;
	3. Передача обработанного значения в микросервис Inventory Service.

ИНФРАСТРУКТУРА:
	БД ПРИЕМКА - временное хранение записанных товаров до окончания сессии приемки.

ТЕХНОЛОГИИ:
	REST API
	gRPC API
	PostgreSQL

---
### 2️⃣ Inventory Service

ОТВЕТСТВЕННОСТЬ:
	1. Хранение информации о товарах на складе;
	2. Запись в БД товаров с приемки;
	3. Списание позиции при продаже;
	4. Пополнение позиции при возврате;
	5. Поиск товара по БД в случае

ИНФРАСТРУКТУРА:
	БД СКЛАД - постоянное хранение записанных товаров.

ТЕХНОЛОГИИ:
	REST API
	gRPC API
	PostgreSQL

---
### 3️⃣ Analytics Service

ОТВЕТСТВЕННОСТЬ:
	1. Формирование итогов по продажам за месяц в виде справки;
	2. Запись в БД истории об операциях и воспроизведение во внутреннем диалоговом окне;

ИНФРАСТРУКТУРА:
	БД ОПЕРАЦИИ - постоянное хранение записанных товаров.

ТЕХНОЛОГИИ:
	REST API
	gRPC API
	PostgreSQL
